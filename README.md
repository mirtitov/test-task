# Простой HTTP-сервер записей

Сервер принимает запросы на сохранение и выдачу записей. Запись — объект из двух полей: **id** (уникальный идентификатор) и **data** (произвольные данные). Между запусками все записи сохраняются в JSON-файл (без СУБД). Зависимости — только стандартная библиотека Python 3.

## Запуск

```bash
cd simple_record_server
python server.py
```

Сервер слушает порт **8080** на всех интерфейсах (`http://0.0.0.0:8080`). Остановка: `Ctrl+C`.

Создаётся каталог `data/` и файл `data/records.json` при первой записи.

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/records` | Сохранить запись. Тело: `{"id": "<уникальный_id>", "data": <любые_данные>}` |
| GET | `/records/<id>` | Получить одну запись по идентификатору |
| GET | `/records` | Получить все записи |
| GET | `/records?ids=id1,id2,...` | Получить выбранные записи по списку id |

Ответы в формате JSON. Коды: 200 (OK), 201 (Created), 400 (ошибка запроса), 404 (не найдено).

## Проверка работоспособности

### Вручную (curl)

```bash
# Сохранить запись
curl -X POST http://127.0.0.1:8080/records \
  -H "Content-Type: application/json" \
  -d '{"id": "test-1", "data": {"name": "Example", "value": 42}}'

# Получить одну запись
curl http://127.0.0.1:8080/records/test-1

# Все записи
curl http://127.0.0.1:8080/records

# Несколько по id
curl "http://127.0.0.1:8080/records?ids=test-1,other-id"
```

### Скрипт оценки производительности

В другом терминале (сервер должен быть запущен):

```bash
cd simple_record_server
python perf_test.py
```

Можно указать базовый URL:

```bash
python perf_test.py http://127.0.0.1:8080
```

Скрипт выводит количество операций, время и ориентировочные запросы в секунду для последовательных и параллельных сценариев, а также для выдачи множества записей.
